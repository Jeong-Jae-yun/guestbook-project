# AWS 배포 설명

## 1. EC2를 선택한 이유

AWS EC2는 서버 환경을 직접 구성하고 제어할 수 있어 Docker 기반 배포 과정을 학습하기에 적합하다. 또한 비교적 간단한 설정으로 퍼블릭 IP를 통한 서비스 공개가 가능하여 개인 프로젝트 및 학습용 배포 환경으로 적절하다고 판단하였다.

## 2. 보안 그룹 설정 이유

- **22번 포트(SSH)** : 로컬 PC에서 EC2 인스턴스로 접속하기 위해 허용하였다.
- **3000번 포트(Frontend)** : 브라우저에서 프론트엔드 서비스에 접근하기 위해 개방하였다.
- **8080번 포트(Backend)** : REST API 서버(Spring Boot)에 접근하기 위해 개방하였다.
  필요한 포트만 최소한으로 개방하여 보안을 유지하도록 설정하였다.

## 3. 서버에서 실행한 명령 흐름

1. EC2 인스턴스 생성 및 보안 그룹 설정
2. SSH를 통해 EC2 접속
3. 프로젝트 클론
4. Docker 및 Docker Compose 설치
5. docker-compose up 명령으로 컨테이너 실행

## 4. 배포 후 접속 방식

- Frontend: `http://EC2_PUBLIC_IP:3000`
- Backend API: `http://EC2_PUBLIC_IP:8080/api/guestbooks`
  브라우저를 통해 프론트엔드에 접속하고, 프론트엔드에서 백엔드 API를 호출하는 방식으로 서비스가 동작한다.
